# Canvas Magic - AI 驱动的设计生成系统

## 项目概述

Canvas Magic 是一个长期学习项目，旨在通过 AI 代理从高质量 HTML 示例中学习，提炼出精美的 Canvas/Three.js/Shader 效果，最终让用户用一句话就能生成专业级的视觉设计。

## 项目结构

```
canvas-magic/
├── AGENTS.md           # 本文件 - 项目使用说明和系统关系
├── SOULS.md            # AI 代理的主动学习能力框架
├── MEMORY.md           # 限定长度的必要知识记忆
├── v0.1/               # 版本 0.1 - 初始技能版本
│   ├── SKILL.md        # 技能主文件
│   └── references/     # 参考资料
├── v0.2/               # 版本 0.2 - 改进版本（待创建）
├── v0.3/               # 版本 0.3 - 更多效果（待创建）
├── changelogs/         # 版本复盘和变更日志
│   ├── v0.1-review.md  # v0.1 复盘
│   └── v0.2-review.md  # v0.2 复盘
└── shareds/            # 跨版本共享资源
    ├── test-cases/     # 测试用例
    ├── patterns/       # 提炼的设计模式
    └── knowledge/      # 积累的知识点
```

## 核心文件关系

### 1. AGENTS.md（本文件）
- **作用**：项目入口，解释系统如何工作
- **内容**：项目结构、使用方法、文件关系
- **更新频率**：项目结构变化时更新

### 2. SOULS.md
- **作用**：定义 AI 代理的"灵魂" - 主动学习能力
- **内容**：学习策略、实验方法、迭代逻辑
- **灵感来源**：OPENCLAW、HERMES-AGENTS 的主动学习理念
- **更新频率**：学习策略改进时更新

### 3. MEMORY.md
- **作用**：维护限定长度的必要知识记忆
- **内容**：关键发现、最佳实践、错误教训
- **特点**：保持简洁，定期清理过时信息
- **更新频率**：每次学习后更新

### 4. 版本文件夹（v0.1, v0.2, ...）
- **作用**：存储特定版本的技能和产出
- **内容**：SKILL.md（技能定义）+ references（参考资料）
- **版本策略**：语义化版本（主版本.次版本）

### 5. changelogs/
- **作用**：记录每个版本的复盘和改进
- **内容**：成功经验、失败教训、改进建议
- **格式**：Markdown，包含日期和版本号

### 6. shareds/
- **作用**：跨版本共享的资源积累
- **内容**：测试用例、设计模式、知识点
- **特点**：长期有效，不随版本变化

## 使用方法

### 快速开始
```bash
# 1. 进入项目目录
cd ~/.claude/skills/canvas-magic

# 2. 查看当前版本
ls v0.1/

# 3. 使用技能（在 opencode 中）
skill canvas-magic
```

### 学习流程
1. **输入**：用户的一句话描述
2. **匹配**：SOULS.md 定义的学习策略选择模式
3. **生成**：基于 MEMORY.md 中的知识生成 HTML
4. **输出**：保存到当前版本文件夹
5. **复盘**：记录到 changelogs/
6. **记忆**：更新 MEMORY.md

### 版本升级流程
1. 创建新版本文件夹（如 v0.2/）
2. 复制上一版本的 SKILL.md
3. 根据复盘改进技能
4. 更新 MEMORY.md
5. 记录到 changelogs/v0.2-review.md
6. 提交 Git

## Git 管理策略

### 分支策略
- `main`：稳定版本
- `develop`：开发中的版本
- `feature/*`：新功能分支

### 提交规范
```

类型：
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具
```

### 版本标签
- `v0.1`：初始版本
- `v0.2`：改进版本

## 质量保证

### 生成质量检查
- [ ] 单文件 HTML，无外部依赖
- [ ] 响应式设计，支持移动端
- [ ] 60fps 流畅动画
- [ ] 鼠标交互支持
- [ ] 和谐的颜色方案

### 测试方法
1. 在浏览器中打开生成的 HTML
2. 检查视觉效果
3. 测试鼠标交互
4. 验证响应式布局
5. 检查性能（DevTools）
